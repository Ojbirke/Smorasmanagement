Backup Type Information
Git Deployment Backup

This is the most reliable option for restoring after redeployments. It contains the data that was saved to Git during the last deployment process.

Deployment Backups

These backups are created specifically for production deployments and contain all the essential data needed for the application to function properly.

Persistent Backups

These backups are designed to survive across deployments. They are stored in a location that is not affected by code deployments.

Regular Backups

Standard application backups created during normal operation. These may be overwritten during deployments.

Emergency Recovery

This option will recreate essential system data from scratch. Use only as a last resort when no backups are available.
