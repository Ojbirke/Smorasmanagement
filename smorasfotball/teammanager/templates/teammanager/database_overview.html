Database Management Instructions
Creating Backups

Two types of backups are available:

  • Regular Backup: Creates a standard backup that is stored in both regular and persistent locations.
  • Deployment Backup: Creates a special backup designed for deploying the application to production. This is used when the application is redeployed.
Restoring Backups

Click the "Restore" button next to a backup file to restore the database to that state. Warning: This will replace all current data in the database with the data from the backup.

Backup Types
  • JSON Data: Contains all data in a format that can be loaded into any database backend. Best for transferring data between environments.
  • SQLite Database: Direct copy of the SQLite database file. Contains all data including system tables. Only works with SQLite.
Deployment Backups

Deployment backups are special backups used when deploying the application to production:

  • Location: Stored in the /deployment directory
  • Purpose: Used during application deployment to ensure the production environment has the correct database state
  • Active File: deployment_db.json is the specific file used during deployment

When you click "Create Deployment Backup", a timestamped backup is created and also linked as deployment_db.json.

Persistent Backups

When you create a regular backup, it's automatically stored in two locations:

  • Regular location: The /backups folder inside the application directory
  • Persistent location: A special /persistent_backups folder that remains available after redeployments
Automatic Restoration

The system will automatically detect and restore from the latest persistent backup when:

  • The application is redeployed
  • The database is empty or not initialized

This ensures your data is preserved across redeployments without manual intervention.

Manual Restoration

You can also manually restore from a persistent backup:

  1. Use the Restore button in the Persistent Backups section above
  2. Or use this command in the terminal:
cd smorasfotball && python manage.py restore_persistent_backup your_backup_filename.json
Important: Always create a backup before making major changes to the system or data.
