Recovery Process Information
What Happens During Recovery
  1. A backup of your current database is created as a precaution
  2. Your current database is completely replaced with the selected backup
  3. Database migrations are run to ensure schema compatibility
  4. The application is restarted to apply the changes
Note: The recovery process may take a few moments to complete.
